Defect Severity Matrix
TRIAGE
CRITICAL
System crash, data loss, security breach. No workaround.
HIGH
Major feature broken. Workaround difficult or unavailable.
MEDIUM
Feature partially broken. Workaround exists.
LOW
Cosmetic or minor UX issue. Does not impact functionality.
PRIORITY MAPPING
CRITICAL → Fix immediately (P0)BLOCKER
HIGH → Fix in current sprint (P1)URGENT
MEDIUM → Fix in next sprint (P2)NORMAL
LOW → Backlog / nice to fix (P3)MINOR
△
Testing Pyramid
STRATEGY
E2E
Slow · Expensive · Few
INTEGRATION
Medium speed · Service-level
UNIT TESTS
Fast · Cheap · Many
Rule of thumb: 70% unit / 20% integration / 10% E2E. Invert the pyramid and you'll have a flaky, slow test suite.

# Selenium WebDriver — Python with Page Object Model from selenium import webdriver from selenium.webdriver.common.by import By import unittest class LoginPage: def __init__(self, driver): self.driver = driver self.EMAIL = (By.ID, "email") self.PASSWORD = (By.ID, "password") self.SUBMIT = (By.CSS_SELECTOR, "button[type='submit']") def login(self, email, password): self.driver.find_element(*self.EMAIL).send_keys(email) self.driver.find_element(*self.PASSWORD).send_keys(password) self.driver.find_element(*self.SUBMIT).click()

Testing Types Reference
REFERENCE
🔬
FUNCTIONAL
Validates software against requirements. Tests what the system does.
⚡
PERFORMANCE
Load, stress, spike & endurance testing. Measures speed, stability, and scalability.
🔐
SECURITY
Pen testing, OWASP checks, auth testing. Finds vulnerabilities before attackers do.
♿
ACCESSIBILITY
WCAG compliance, screen reader compatibility, keyboard navigation.
📱
COMPATIBILITY
Cross-browser, cross-device, cross-OS testing. Ensures consistent experience.
🔄
REGRESSION
Verifies new changes don't break existing functionality. Run after every build.
💨
SMOKE
Quick sanity check on critical paths. First gate before deeper testing.
🤝
UAT
End-users validate the system meets business needs before go-live.
